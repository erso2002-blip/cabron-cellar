## Report Control

Endpoint:

- `GET /api/waitlist/report`

Security:

- Requires `WAITLIST_REPORT_TOKEN` in production.
- Requires request header `x-waitlist-report-token`.
- Without a configured/matching token, returns `404`.

Main metrics:

- `requested`: leads who submitted the waitlist form.
- `loggedIn`: leads whose e-mail appears in the app users table.
- `pending`: requested minus loggedIn.
- `conversionRate`: loggedIn divided by requested.
- `sources`: same metrics grouped by campaign/source.

## Rollback

- Roll back app deploy to `v0.2.17` if the service-worker behavior causes unexpected refresh issues.
- The report endpoint is additive and does not require database migration.
