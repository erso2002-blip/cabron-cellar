# v0.2.0 Candidate

## Status

- Public app version target: `0.2.0`
- Production status: pending deploy.
- Rollback base: `v0.1.0` / commit `b22f29f`.

## Included Changes

- Adds public `/waitlist` page and `/api/waitlist` lead capture endpoint.
- Adds optional winery website field for wine create/edit/detail.
- Adds passwordless email login with six-digit code and database-backed sessions.
- Keeps Google SSO available as the primary login method.

## Required Production Setup

- Apply database schema for:
  - `waitlist_leads`
  - `email_login_codes`
  - `email_sessions`
  - `wines.winery_website_url`
- Configure email delivery before relying on email login:
  - `RESEND_API_KEY`
  - optional `AUTH_EMAIL_FROM`
  - optional `EMAIL_LOGIN_PEPPER`

## Validation Checklist

- `pnpm run typecheck`
- `pnpm --filter @workspace/api-server build`
- `PORT=3000 BASE_PATH=/ pnpm --filter @workspace/minha-adega build`
- Public `/api/healthz` returns 200 after deploy.
- Public `/api/auth/config` shows Google configured and email login status.
- Public `/waitlist` returns 200.
- Manual login test with Google.
- Manual email login test after email provider is configured.

## Rollback

- Prefer Vercel rollback to the previous `v0.1.0` deployment if the issue is runtime/UI.
- Do not drop nullable/new tables unless a schema issue directly requires it; `v0.1.0` should ignore extra schema.
