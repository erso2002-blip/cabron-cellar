# v0.2.53

## Summary

- Add Mercado Pago subscription persistence for checkout creation.
- Add Mercado Pago webhook endpoint for subscription preapproval reconciliation.
- Add authenticated billing status endpoint.
- Gate Pro access through either approved beta override or active persisted subscription.
- Keep paid buttons disabled in the front end until the controlled end-to-end billing test is approved and completed.

## Validation

- `pnpm --filter @workspace/api-server typecheck`

## Production Status

- Prepared for controlled production deployment.
- Billing remains blocked for public use until Vercel webhook envs are configured and a sandbox/test-user flow confirms:
  - checkout creation
  - local pending subscription record
  - webhook receipt
  - Mercado Pago status reconciliation
  - Pro access only after authorized status

## Rollback

- Roll back app deploy to `v0.2.52` if the billing schema bootstrap, webhook route, or Pro gate causes unexpected production behavior.
- Schema additions are additive: `billing_subscriptions` and `billing_webhook_events`.
