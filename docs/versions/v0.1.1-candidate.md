- Candidate: yes
- Public app version target: `0.1.1`

- Add optional `Site da viticola` field to wine create/edit form.
- Save winery website as `wineryWebsiteUrl` through API and schema.
- Display winery website as an external link in wine detail.

- Typecheck passed locally.
- Frontend build passed locally.
- API build passed locally.

## Production Requirements

- Bump app/API/package/OpenAPI versions from `0.1.0` to `0.1.1`.
- Apply database column `winery_website_url` in production using the correct production `DATABASE_URL`.
- Deploy to Vercel.
- Validate create, edit, detail, and list flows with Google SSO active.
- Confirm older wine records without winery website still render correctly.

## Rollback

- Preferred rollback before release: do not deploy candidate.
- If released and issue is UI/API-only: rollback Vercel to `v0.1.0`.
- If released with database column already added: leave the nullable column in place unless there is a specific data/schema issue. The `v0.1.0` code should ignore the extra column.

## Known Risk

- The candidate includes a production database schema change. Because the column is nullable, rollback should be low risk, but schema application must be done against the correct production database only.
