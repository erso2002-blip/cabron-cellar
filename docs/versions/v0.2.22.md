# v0.2.22

## Changes

- Add the authenticated `Assinatura` screen with Free, Pro Mensal, and Pro Anual plans.
- Add `GET /api/billing/plans` and `POST /api/billing/checkout`.
- Create Mercado Pago recurring checkout sessions through `/preapproval` when `MERCADO_PAGO_ACCESS_TOKEN` is configured.
- Configure production environment variables `MERCADO_PAGO_ACCESS_TOKEN` and `MYCELLAR_PUBLIC_URL` in Vercel.
- Bump app version to `0.2.22` and service worker cache to `minha-adega-v20`.

## Validation

- Local `pnpm --filter @workspace/api-server typecheck` passed.
- Local `pnpm --filter @workspace/minha-adega typecheck` passed.
- Local `pnpm --filter @workspace/api-server build` passed.
- Local `PORT=3000 BASE_PATH=/ pnpm --filter @workspace/minha-adega build` passed.

## Known Gap

- Automatic Pro activation still requires subscription persistence and Mercado Pago webhook/status reconciliation before public paid sales.

## Rollback

- Roll back app deploy to `v0.2.21` if the billing screen or checkout endpoint causes unexpected production behavior.
